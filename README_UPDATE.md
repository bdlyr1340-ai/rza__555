# تحديث: تشخيص تحديات Google + معالجة موسّعة

  ## ما الذي تغيّر؟
  ملفان فقط — استبدلهما في مشروعك على Railway:

  1. `bot/handlers/start.py`
  2. `bot/services/sheerid.py`

  ## كيف تطبّق التحديث؟

  ### Railway (الأسهل):
  1. فك ضغط هذا الملف.
  2. ادخل GitHub repo المربوط بـ Railway.
  3. استبدل الملفين فوق ملفاتك القديمة.
  4. Commit + Push → Railway يعيد النشر تلقائياً.

  ---

  ## المشكلة التي تحلّها هذه النسخة
  وصلتك الرسالة:
  ```
  [GOOGLE_CHALLENGE]
  💡 Google يطلب التحقق من الهوية...
  ```
  هذا يعني أن البوت **نجح** في إدخال الإيميل وكلمة السر، لكن Google عرض صفحة "تحقق من هويتك"
  ولم نعرف بالضبط ماذا طلب: إيميل احتياطي؟ هاتف؟ تأكيد جهاز؟

  ## الإصلاحات الجديدة

  ### 1) معالج تحديات أقوى 🤖
  كان يحاول زر **"Yes"** فقط. الآن يجرّب أيضاً:
  - Continue / Next / Confirm / OK
  - That's me / It was me / Yes, it was
  - متابعة / التالي / تأكيد / موافق / نعم

  ### 2) تخطّي الإشعارات الناعمة (لا فشل بعد الآن)
  عندما يطلب Google "أضف رقم هاتف" أو "احفظ هذا الجهاز" → البوت يضغط
  **Skip / Not now / ليس الآن / تخطّى** ويكمل تلقائياً بدلاً من الفشل.

  ### 3) تشخيص أوضح
  الآن إذا فشل التحدي، الكود يعرف نوعه ويُبلِّغ:
  - `Recovery email required` — يطلب إيميل استرداد
  - `Phone verification required` — يطلب رقم هاتف
  - `Device-tap required` — يطلب الموافقة من جهاز آخر
  - `Unknown challenge` — نوع غير معروف (تُرفق له لقطة)

  ### 4) **لقطة شاشة + HTML كامل للأدمن** 📸
  هذا أهم تحسين. عند فشل أي تحدي، البوت يحفظ **لقطة شاشة كاملة** للصفحة
  + نص الصفحة + ملف HTML الكامل، ويرسلها لك أنت كأدمن مع رسالة الفشل العادية.

  عندك ستصلك **3 رسائل** لكل فشل تحدي:
  1. الرسالة النصية (المستخدم/الطلب/السبب) كالعادة.
  2. **📸 لقطة الصفحة** — تشاهد بعينيك ماذا عرض Google.
  3. **🔍 تفاصيل** نصية (URL + العنوان + أول 500 حرف من الصفحة).
  4. **📄 ملف HTML كامل** — مرفق يمكنك فتحه في المتصفح لو احتجنا تحليل أعمق.

  بهذا الشكل: عندما تفشل عملية، أرسل لي اللقطة + التفاصيل وسأعرف بالضبط
  ماذا يطلبه Google ونصلحه معاً.

  ---

  ## ملاحظة سريعة
  الـ `GOOGLE_CHALLENGE` على سيرفرات Cloud (مثل Railway) شائع جداً لأن Google
  لا يثق بـ IPs مراكز البيانات. الحل الجذري: **بروكسي سكني (Residential)** —
  مكتوب في `.env.example` كيف تضيف `PROXY_LIST`.

  لكن أولاً جرّب هذه النسخة وأرسل لي اللقطات لو فشلت — قد نتمكن من تخطّي
  بعض الحالات بدون بروكسي.
